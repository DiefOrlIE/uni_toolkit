01

What is HTML?

HTML stands for HyperText Markup Language. It is not a programming language — it has no logic, no loops, no variables. It is a markup language: a way to annotate text so a browser knows how to display it.

CS mental model Think of HTML as a data structure — specifically a tree (the DOM). Every element is a node. Parent/child relationships define nesting. Your browser parses this tree and renders it visually.

The browser reads your HTML file top-to-bottom, builds a tree called the DOM (Document Object Model), and paints pixels on screen. CSS styles those pixels. JavaScript manipulates the tree at runtime.

HTML
Structure & content. What exists on the page.
CSS
Style & layout. What things look like.
JavaScript
Behavior & logic. What things do.
02

Document Structure

Every HTML file follows this skeleton. Memorize it — you'll type it thousands of times.

skeleton.html 
<!-- This line tells the browser: "I'm HTML5" -->
<!DOCTYPE html>

<html lang="en">

  <head>
    <!-- Metadata: not visible on the page -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Page Title</title>
  </head>

  <body>
    <!-- Visible content goes here -->
    <h1>Hello, World!</h1>
  </body>

</html>

Key points

PartRole
<!DOCTYPE html>Tells browser to use HTML5 (not optional)
<html>Root element — every other element is its descendant
<head>Metadata, links to CSS/JS files, page title
<body>Everything the user sees
03

Core Tags

HTML has ~100 tags. You'll use maybe 20 regularly. Here are the essential ones.

Text

text-tags.html
<!-- Headings: h1 is biggest, h6 is smallest -->
<h1>Page Title</h1>
<h2>Section</h2>
<h3>Sub-section</h3>

<!-- Paragraph -->
<p>This is a paragraph of text.</p>

<!-- Inline: bold, italic, code -->
<p>This is <strong>bold</strong>, this is <em>italic</em>.</p>
<p>Inline code: <code>print("hello")</code></p>

<!-- Line break (no closing tag needed) -->
<br>

<!-- Horizontal rule -->
<hr>

Containers

containers.html
<!-- div: block-level generic container -->
<div>
  <p>I'm inside a div.</p>
</div>

<!-- span: inline generic container -->
<p>Normal text <span style="color:red">red text</span> normal again.</p>
Block vs Inline <div> is block-level: takes the full width, starts on a new line. <span> is inline: flows with text. This distinction matters hugely for CSS layout.
04

Attributes

Attributes provide extra information about an element. Syntax: name="value" inside the opening tag.

attributes.html
<!-- id: unique identifier (like a variable name) -->
<p id="intro">First paragraph.</p>

<!-- class: reusable group label -->
<p class="highlight">Styled paragraph.</p>

<!-- style: inline CSS (avoid for large projects) -->
<p style="color: blue; font-size: 20px;">Blue text.</p>

<!-- title: tooltip on hover -->
<abbr title="HyperText Markup Language">HTML</abbr>
id vs class id must be unique per page — like a primary key in a database. class can be shared by many elements. In JavaScript: getElementById() vs getElementsByClassName().
05

Links & Images

links-images.html
<!-- Anchor tag: href = where to go -->
<a href="https://example.com">Click me</a>

<!-- Open in new tab -->
<a href="https://example.com" target="_blank">New tab</a>

<!-- Relative link (same website) -->
<a href="./about.html">About page</a>

<!-- Jump to element by id -->
<a href="#intro">Go to intro</a>

<!-- Image: self-closing, alt is required for accessibility -->
<img src="photo.jpg" alt="A description" width="300">

URL types

TypeExampleWhen to use
Absolutehttps://site.com/pageExternal links
Relative./images/logo.pngFiles within your project
Root-relative/images/logo.pngFrom site root
Fragment#section-idJump within same page

Lists & Tables

lists.html
<!-- Unordered list (bullets) -->
<ul>
  <li>Python</li>
  <li>C++</li>
  <li>JavaScript</li>
</ul>

<!-- Ordered list (numbered) -->
<ol>
  <li>First</li>
  <li>Second</li>
</ol>

<!-- Table structure -->
<table>
  <thead>
    <tr>
      <th>Name</th>  <!-- th = header cell -->
      <th>Grade</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>  <!-- td = data cell -->
      <td>A</td>
    </tr>
  </tbody>
</table>
07

Forms

Forms collect user input and send it to a server (or process it with JavaScript). Very important for web apps.

form.html
<form action="/submit" method="POST">

  <!-- Text input -->
  <label for="username">Username:</label>
  <input type="text" id="username" name="username" placeholder="Enter name">

  <!-- Password -->
  <input type="password" name="pwd">

  <!-- Dropdown -->
  <select name="major">
    <option value="cs">Computer Science</option>
    <option value="ee">Electrical Engineering</option>
  </select>

  <!-- Checkbox -->
  <input type="checkbox" name="agree" id="agree">
  <label for="agree">I agree</label>

  <!-- Submit button -->
  <button type="submit">Submit</button>

</form>
GET vs POST method="GET" appends data to the URL (visible, bookmarkable — use for search). method="POST" sends data in the request body (use for login, file upload, sensitive data).
08

Semantic HTML

Instead of using <div> for everything, HTML5 introduced semantic tags that describe meaning. This matters for accessibility, SEO, and readability.

semantic.html
<body>
  <header>
    <nav>...</nav>
  </header>

  <main>
    <article>
      <h1>Post Title</h1>
      <section>
        <p>Content...</p>
      </section>
    </article>

    <aside>Related links</aside>
  </main>

  <footer>© 2026</footer>
</body>
<header>
Top of page or section
<nav>
Navigation links
<main>
Primary content (one per page)
<article>
Self-contained content (blog post)
<section>
Thematic grouping
<aside>
Sidebar / tangential content
<footer>
Bottom of page or section
09

HTML + CSS + JS Together

Here's how all three layers connect in one file:

complete-example.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Counter</title>

  <!-- CSS: <style> tag in <head> -->
  <style>
    body { font-family: sans-serif; text-align: center; padding: 40px; }
    #count { font-size: 3rem; color: #7aa2f7; }
    button { padding: 10px 24px; font-size: 1rem; cursor: pointer; }
  </style>
</head>
<body>

  <!-- HTML structure -->
  <h1>Click Counter</h1>
  <p id="count">0</p>
  <button onclick="increment()">Click me</button>

  <!-- JS: <script> tag before </body> -->
  <script>
    let n = 0;
    function increment() {
      n++;
      document.getElementById('count').textContent = n;
    }
  </script>

</body>
</html>
live preview
How to link external files CSS: <link rel="stylesheet" href="style.css"> in <head>
JS: <script src="app.js"></script> before </body>
