01

What is HTML?

HTML stands for HyperText Markup Language. It is not a programming language — it has no logic, no loops, no variables. It is a markup language: a way to annotate text so a browser knows how to display it.

CS mental model Think of HTML as a data structure — specifically a tree (the DOM). Every element is a node. Parent/child relationships define nesting. Your browser parses this tree and renders it visually.

The browser reads your file top-to-bottom, builds the DOM (Document Object Model), and paints pixels. CSS styles those pixels. JavaScript manipulates the tree at runtime.

HTML
Structure & content. What exists on the page.
CSS
Style & layout. What things look like.
JavaScript
Behavior & logic. What things do.
02

Document Structure

Every HTML file follows this skeleton. Memorize it — you'll type it thousands of times.

skeleton.html 
<!-- Tells the browser: "I am HTML5" -->
<!DOCTYPE html>

<html lang="en">

  <head>
    <!-- Metadata: not visible on the page -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Page Title</title>
  </head>

  <body>
    <!-- Visible content goes here -->
    <h1>Hello, World!</h1>
    <p>This is my first HTML page.</p>
  </body>

</html>
PartRole
<!DOCTYPE html>Declares HTML5 — not optional
<html>Root element; every other element is its descendant
<head>Metadata, CSS/JS links, page title (invisible)
<body>Everything the user actually sees

Containers: div vs span

containers.html 
<!-- div: block-level generic container -->
<div>I am a div (block). I take full width.</div>
<div>Another div — on a new line.</div>

<!-- span: inline, flows with text -->
<p>Normal <span style="color:red">red text</span> normal again.</p>
Block vs Inline <div> is block-level: starts on a new line, takes full width. <span> is inline: sits inside text. This distinction is fundamental to CSS layout.
04

Attributes

Attributes add extra information to elements. Syntax: name="value" inside the opening tag.

attributes.html 
<!-- id: unique identifier -->
<p id="intro">I have id="intro".</p>

<!-- class: reusable group label -->
<p class="highlight">Styled paragraph.</p>

<!-- style: inline CSS -->
<p style="color:blue; font-size:20px;">Blue text.</p>

<!-- title: tooltip -->
<abbr title="HyperText Markup Language">HTML</abbr>
id vs class id must be unique per page — like a primary key. class can be shared. In JS: getElementById() vs getElementsByClassName().

07

Forms

Forms collect user input and send it to a server (or handle it with JavaScript).

form.html 
<form action="/submit" method="POST">

  <label for="username">Username:</label>
  <input type="text" id="username" placeholder="Enter name">

  <select name="major">
    <option value="cs">Computer Science</option>
    <option value="ee">Electrical Engineering</option>
  </select>

  <input type="checkbox"> I agree
  <button type="submit">Submit</button>

</form>
GET vs POST GET appends data to the URL (visible — use for search). POST sends data in the body (hidden — use for passwords, sensitive data).
08

Semantic HTML

Instead of <div> for everything, HTML5 added semantic tags that describe meaning. Better for accessibility, SEO, and readability.

semantic.html 
<header>
  <nav>...</nav>
</header>

<main>
  <article>
    <h1>Post Title</h1>
    <section><p>Content...</p></section>
  </article>
  <aside>Related links</aside>
</main>

<footer>© 2026</footer>
<header>
Top of page or section
<nav>
Navigation links
<main>
Primary content (one per page)
<article>
Self-contained content
<section>
Thematic grouping
<aside>
Sidebar content
<footer>
Bottom of page
09

HTML + CSS + JS Together

Here's how all three layers connect — a fully working interactive counter:

complete-example.html 
<!DOCTYPE html>
<html lang="en">
<head>
  <title>Counter</title>
  <style>
    body { text-align: center; padding: 30px; }
    #count { font-size: 4rem; color: #7aa2f7; }
    button { padding: 10px 24px; cursor: pointer; }
  </style>
</head>
<body>
  <h1>Click Counter</h1>
  <p id="count">0</p>
  <button onclick="increment()">Click me</button>
  <script>
    let n = 0;
    function increment() {
      n++;
      document.getElementById('count').textContent = n;
    }
  </script>
</body>
</html>
How to link external files CSS: <link rel="stylesheet" href="style.css"> inside <head>
JS: <script src="app.js"></script> just before </body>
