Study Guide

学习导引

学 Python OOP 最容易遇到的问题，不是不会写类，而是把面向对象写成了“语法版面向过程”。因此这份手册的学习顺序不是“看一眼概念就往后翻”，而是反复问自己：这个对象负责什么？它暴露什么接口？它和别的对象如何协作？

1
先读语法。 每章先把语法规则和最小例子读懂，不要急着扩展。
2
再看误区。 Python 中很多写法“能跑”不代表“适合长期维护”。
3
最后看应用。 游戏建模只是载体，真正要学的是如何拆职责和组织对象。
适合谁

你已经理解变量、函数、循环，也写过 C++ 里的类与对象；但面对 Python 的 self、__init__、双下划线方法、抽象基类时，还没有建立一套稳定的直觉。

读完之后应达到什么状态

你应该能够独立设计一个小型对象系统，并说明每个类为什么存在、公开哪些方法、哪里应该继承、哪里更适合组合。

Chapter 01

为什么要学面向对象

先搞清楚 OOP 在解决什么问题，而不是只记住几个术语。

当程序还很小时，我们常常用函数把流程一路写下去：读取数据、计算结果、输出信息。这种写法在任务单一、步骤稳定时非常高效。但一旦程序开始涉及多个实体、多个状态与多个行为，单纯靠函数堆叠就会让代码越来越像一条失控的流程线。

面向对象的核心思想，是把程序中的重要实体抽象成对象。每个对象有自己的状态，也有自己的行为。这样一来，你思考问题的单位就不再只是“函数下一步做什么”，而是“谁负责这件事”。

面向过程

关注步骤：先做 A，再做 B，再做 C。适合线性任务，但数据容易散落在全局变量和长参数列表中。

面向对象

关注职责：角色有什么属性，能做什么，和谁协作。适合复杂系统建模，也更利于扩展。

一个最朴素的例子

假设你在写一个小游戏。程序里会出现角色、技能、怪物、背包。如果全部写成函数，你会在很多地方传递血量、攻击力、技能冷却时间、背包内容等参数。随着逻辑变多，这些数据会越来越难管理。

常见误区

不是所有程序都必须重度面向对象。面向对象不是“越多类越专业”，而是当系统里确实存在稳定的实体和职责边界时，它能让结构更清楚。

本章小结

学习 OOP 的第一步，不是记住封装、继承、多态，而是理解：当程序开始描述多个实体和它们的协作时，用对象组织代码会比单纯堆函数更自然。

Chapter 02

类与对象入门

类是蓝图，对象是按照蓝图创建出来的具体实例。

在 Python 中，class 用来定义类。类本身描述了对象应该有什么属性、可以执行什么方法；而对象则是这个类的一个具体实例。

class Hero:
    pass

knight = Hero()
mage = Hero()

print(type(knight))
print(knight is mage)

这里定义了一个最简单的 Hero 类，然后创建了两个对象。它们都来自同一个类，但彼此不同，是两份独立的实例。

一句话记忆

class Hero: 是在定义对象的结构；Hero() 是在真正创建一个对象。

class Hero:
    def say_hello(self):
        print("我是一个游戏角色")

hero = Hero()
hero.say_hello()
本章小结

类负责描述，对象负责承载具体状态。以后我们添加的属性和方法，都会围绕“如何让某类对象更完整”来展开。

Chapter 03

属性与方法

对象内部保存什么信息，以及对象能够执行什么行为。

属性描述对象的状态，方法描述对象的行为。对于一个游戏角色来说，姓名、职业、生命值都可以是属性；攻击、移动、使用技能都可以是方法。

class Hero:
    camp = "联盟"

    def __init__(self, name, hp):
        self.name = name
        self.hp = hp

    def attack(self):
        print(f"{self.name} 发动了攻击")

self.name 和 self.hp 是实例属性；camp 是类属性。

C++ 迁移提示

如果你有 C++ 背景，可以把 self 理解成显式写出来的 this。区别在于：Python 要求你在方法定义中把它写出来，而不是像 C++ 那样隐式存在。

class Skill:
    category = "主动技能"

    def __init__(self, name, damage):
        self.name = name
        self.damage = damage

    def cast(self):
        print(f"释放 {self.name}，造成 {self.damage} 点伤害")
常见误区

不要把本应随着实例变化的值写成类属性，否则对象会不合理地共享同一份数据。

本章小结

属性是状态，方法是行为；self 让方法知道自己正在服务哪个对象。

Chapter 04

构造函数与对象初始化

让对象在创建瞬间就进入一个合理、可用的状态。

Python 中最常见的初始化方法是 __init__。它会在对象创建后自动执行，负责把对象需要的初始属性准备好。

class Monster:
    def __init__(self, name, hp, attack):
        self.name = name
        self.hp = hp
        self.attack = attack

slime = Monster("史莱姆", 80, 12)
class Equipment:
    def __init__(self, name, rarity="普通", level=1):
        self.name = name
        self.rarity = rarity
        self.level = level
C++ 迁移提示

__init__ 常被拿来和 C++ 构造函数对比。它们的作用类似，但 Python 的对象已经先被创建，再调用 __init__ 做初始化。

常见误区

不要在对象创建后到处补属性，例如先创建对象、再手动补 name 和 hp。这种做法会让对象在一段时间内处于不完整状态。

本章小结

__init__ 的使命，是确保对象一创建出来就具备可工作的基本状态。

Chapter 05

封装与访问控制

Python 强调约定式封装，而不是像 C++ 那样依赖强制关键字。

封装的目标是让对象把自己的内部细节藏起来，只暴露必要接口。这样外部代码只需要知道“怎么用”，不需要知道“内部怎么实现”。

  • name：公开属性，外部可以直接访问。
  • _name：约定为内部使用，外部最好不要随意碰。
  • __name：触发名称改写，减少外部误用。
class PlayerAccount:
    def __init__(self, username, password):
        self.username = username
        self._login_state = False
        self.__password = password

    def check_password(self, value):
        return value == self.__password
C++ 迁移提示

C++ 里有明确的 public、private、protected。Python 更强调“约定”和“自觉”。

本章小结

封装不是为了神秘，而是为了控制耦合。Python 更依赖命名约定和接口设计能力。

Chapter 06

继承

让子类在复用父类能力的同时，扩展或改写自己的行为。

class Hero:
    def __init__(self, name, hp):
        self.name = name
        self.hp = hp

    def show_status(self):
        print(f"{self.name} 当前生命值：{self.hp}")

class Warrior(Hero):
    def heavy_strike(self):
        print(f"{self.name} 使用重击")

继承适合表达“是一个”的关系。例如，战士和法师都可以看作角色的一种，所以它们可以继承同一个父类。

class Boss(Hero):
    def __init__(self, name, hp, rage):
        super().__init__(name, hp)
        self.rage = rage
常见误区

不要看到代码相似就急着继承。继承表示稳定层次关系；如果只是“拥有某个对象”，组合通常更稳妥。

本章小结

继承的价值在于复用与扩展，但前提是真实存在清晰的层次关系。

Chapter 07

多态与抽象

同一套调用方式，面对不同对象时，可以表现出不同的行为。

class Warrior:
    def attack(self):
        print("战士近战挥砍")

class Mage:
    def attack(self):
        print("法师释放火球")

def battle_start(hero):
    hero.attack()

这里 battle_start 不关心传进来的是谁，只关心这个对象能不能执行 attack()。这就是多态的力量。

from abc import ABC, abstractmethod

class Unit(ABC):
    @abstractmethod
    def attack(self):
        pass
C++ 迁移提示

你可以把抽象基类类比为 C++ 中包含纯虚函数的基类。不同的是，Python 更轻量，也更鼓励按项目复杂度决定是否真的需要这种约束。

本章小结

多态让调用方更通用，抽象让接口边界更清楚。它们的共同目标，是让程序在增加新类型时尽量少改旧代码。

Chapter 08

特殊方法与 Python 风格对象设计

让你定义的对象更像 Python 内置对象一样自然地工作。

class Inventory:
    def __init__(self, items):
        self.items = items

    def __len__(self):
        return len(self.items)

    def __str__(self):
        return f"背包中有 {len(self.items)} 件物品"
  • __str__：定义用户友好的字符串表示。
  • __repr__：定义更偏向调试的字符串表示。
  • __len__：让对象支持 len(obj)。
  • __add__ 等：让对象支持运算符。
常见误区

不要为了炫技而重载大量特殊方法。只有当对象本身真的具备对应语义时，才应该实现这些方法。

本章小结

特殊方法让对象更贴近 Python 语言生态，是从“能写类”走向“会设计 Python 对象”的关键一步。

Chapter 09

实际应用：小游戏对象建模

把前面所有知识点组织成一个可以讲得清楚的小系统。

from abc import ABC, abstractmethod

class CombatUnit(ABC):
    def __init__(self, name, hp):
        self.name = name
        self.hp = hp

    @abstractmethod
    def attack(self):
        pass

class Skill:
    def __init__(self, name, damage):
        self.name = name
        self.damage = damage

    def cast(self, target):
        target.hp -= self.damage
        print(f"{self.name} 命中 {target.name}，造成 {self.damage} 点伤害")

class Inventory:
    def __init__(self):
        self.items = []

    def add_item(self, item):
        self.items.append(item)

class Hero(CombatUnit):
    def __init__(self, name, hp):
        super().__init__(name, hp)
        self.inventory = Inventory()
        self.skills = []

    def attack(self):
        print(f"{self.name} 发起普通攻击")

class Monster(CombatUnit):
    def attack(self):
        print(f"{self.name} 猛烈扑击")
  • 抽象： CombatUnit 定义了战斗单位的共同接口。
  • 继承： Hero 和 Monster 继承共同基类。
  • 组合： 英雄并不是“继承背包”，而是“拥有背包”，因此使用组合。
  • 职责分离： 技能负责伤害逻辑，背包负责物品管理，角色负责组织它们。
为什么这里不是所有东西都靠继承

很多初学者学到继承后，会试图“凡是有关联就继承”。这是不对的。英雄和背包不是“是一个”的关系，而是“拥有一个”的关系，所以组合更自然。

本章小结

当类之间的边界清楚时，程序会自然地变得可扩展。你之后无论加新技能、新怪物还是新装备，改动范围都更可控。

Next Step

学完这份手册后，下一步做什么

如果你已经能够读懂本手册的所有示例，并能自己改写综合案例，那么你已经跨过了“只会照着语法写类”的阶段。接下来更重要的是，用真实练习巩固对象设计感。

1
把综合案例扩展一轮。 例如加入治疗技能、装备系统或多种怪物类型，观察哪些类需要改，哪些类不需要改。
2
练习组合优先于继承。 主动思考“这个关系真的是 is-a 吗，还是 has-a？”
3
学习属性装饰器与数据类。 它们会让你写出更自然的 Python 风格对象。
4
尝试做一个完整小项目。 比如文本冒险、简化版战斗模拟器或装备管理器，把 OOP 用在真实代码组织中。
记住：真正让你成长的，不是“知道 OOP 是什么”，而是能判断什么时候该建类、什么时候不该建类，什么时候该继承、什么时候该组合。
